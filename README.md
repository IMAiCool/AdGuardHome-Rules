# 🛠️ AdGuard 规则合并与分类处理工具

## 📌 项目目标

开发一个命令行工具，通过本地规则文件与上游规则 URL 实现 AdGuard 广告规则的自动化合并、分类、去重和标准化处理，并输出适用于 AdGuard 的规则文件。

* * *

## 📁 输入文件说明

| 文件名 | 说明  |
| --- | --- |
| `urls.txt` | 上游规则列表，每行一个 URL，支持 HTTP/HTTPS。 |
| `local_rules.txt` | 本地自定义规则文件。以 `@@` 开头的行为白名单规则，其余为黑名单规则。 |

* * *

## 📤 输出文件说明

| 文件名 | 内容描述 |
| --- | --- |
| `all_rules.txt` | 所有上游规则不去重合并结果（原始）。 |
| `blacklist.txt` | 黑名单规则，已标准化、去重并添加 `$important`。 |
| `whitelist.txt` | 白名单规则，已标准化、去重并添加 `$important`。 |
| `blacklist_css.txt` | 含特殊字符的黑名单规则，未纳入去重处理。 |
| `whitelist_css.txt` | 含特殊字符的白名单规则，未纳入去重处理。 |

* * *

## 🔄 处理流程说明

### 1️⃣ 下载与合并上游规则

* 读取 `urls.txt` 中的所有链接；
* 下载并合并所有规则；
* 不去重、不分类，保留注释与空行；
* 保存为 `all_rules.txt`；

### 2️⃣ 特殊字符过滤

* 若规则包含以下任意字符 `/ = \ # ? ( )`（其中 `#` 非首字符），则视为特殊字符规则；
* 特殊字符规则不纳入合并去重分类；
* 以 `@@` 开头的保存到 `whitelist_css.txt`，其余保存到 `blacklist_css.txt`；
* 同时从 `all_rules.txt` 剔除这些规则；

### 3️⃣ 黑白名单初步分类

* 非特殊字符规则按前缀判断归类：
  * 以 `@@` 开头 → 白名单；
  * 其余 → 黑名单；
* 本地规则 `local_rules.txt` 同样按此方式分类；
* 若本地规则未出现在已有名单中，则加入对应名单；

### 4️⃣ 域名提取与冲突处理

* 将规则转换为域名格式（如 `@@||example.com^` → `example.com`）；
* 对黑白名单分别去重；
* 冲突解决策略：
  * 同一域名同时出现在黑白名单中 → 保留白名单，删除黑名单；
  * 黑名单中若有子域名，白名单中对应上级域名应被剔除；

### 5️⃣ AdGuard 格式标准化

* 黑名单格式：`||domain.com^$important`
* 白名单格式：`@@||domain.com^$important`

### 6️⃣ 添加文件元信息头部

在 `blacklist.txt` 和 `whitelist.txt` 文件顶部添加以下信息：

    ! 来源: [所有上游 URL 列表]
    ! 黑名单条目数量: [xxx]
    ! 来自本地规则条目数量: [yyy]
    ! 更新时间: [当前时间]
    ! 更新频率: 每12小时
